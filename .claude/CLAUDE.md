Проект: Скрипт развертывания скрытого прокси на удаленном сервере

---

## Модель общения

АИ и пользователь не имеют прямого доступа к локальным репозиториям друг друга.
Единственная точка синхронизации — удалённый репозиторий.

- Пользователь вносит правки локально → коммит → пуш в удалённый репо.
- АИ вытягивает удалённый репо → генерирует код в нужном формате → пользователь применяет локально.
- АИ **не может пушить**. Репо для АИ — только на чтение.

---

## Цикл работы

### Каждое сообщение (первое или последующее в сессии):

1. **Синхронизация** — хук автоматически вытягивает удалённый репо (`REPO_SYNC` в system-reminder). AI не пушит и не напоминает об этом — пуш полностью запрещён (см. `red-lines.md`).
2. **Загрузка документации** — обязательно прочитать:
   - `.claude/rules/red-lines.md`
   - `.claude/rules/common-mistakes.md`
   - `.claude/REFERENCE.md`
   - `.claude/tasks/<модуль>.md` — если задача касается конкретного модуля
3. **Скептик-Агент** — запустить агент-скептик.
   - Атакует задачу с трёх сторон, **всегда** создает интерактивную Asked-форму и останавливает выполнение.
   - **После** ответов пользователя возвращает `REFINED` — использовать как постановку задачи вместо оригинала, либо `STOP` — завершить без кодогенерации.
4. **Скрытая генерация кода** — создание единого файла-патча (в формате .md) разделенного комментариями к какому файлу относится правка для передачи его в агента.
5. **Слепой ревью** — запустить Agent и передать ему новый "патч.md", правила, документацию (без задачи пользователя).
6. **Финал** - предоставить пользователю способом 1 либо 2 код и правки для переноса в локальное репо.

После этого: пользователь применяет код локально → коммит → пуш → следующий цикл.

## Генерация кода

### Способ 1 — полный файл (на выбор AI от сложности правок)
Новый файл **или** более двух правок на файл:
- вывести файл целиком в блоке кода
- пред блоком кода — путь к файлу (начиная с папки аддона)

### Способ 2 — замена фрагмента (на выбор AI от сложности правок)
До 5 строк, не более двух правок на файл:
- первый блок — старый код
- второй блок — новый код
- пред блоком старого кода — путь к файлу и номер строки в файле

### Способ 3 - создание патча (строго по запросу пользователя).
Файл генерируется и отправляется через чат пользователю.
- Описание коммита: тэги [task-xxx] [bug-xxx] [ref-xxx]
- Себя указывать как контрибьютера.

## Скептик-Агент

Вызвать Agent с model: "haiku" **до любой кодогенерации**.
- [ ] **атакует задачу — не помогает**
- [ ] передать:
- задачу пользователя
- `.claude/REFERENCE.md`
- `.claude/rules/architecture.md`
- реализованный код файлов которые будут затронуты

Агент атакует задачу с трёх сторон:
1. **Нужно ли это вообще?** — ищет готовое решение, похожую реализацию, решение конфигом в `.claude\reference\`*.
2. **Ты просишь то, что хочешь?** — запрашивает явную причину изменений.
3. **Где граница?** — предлагает MVP ≤ 30 строк. «Вот что будет. Хочешь больше — обоснуй.»

По каждой атаке формирует вопрос. **Всегда** выводит **интерактивную asked-форму** и **останавливает выполнение до ответа** пользователя:
```
questions: [{"header":"","multiSelect":false,"options":[{"label":"","description":""}],"question":""}]
```

Ожидаемый ответ агента (после ответов пользователя):
- `REFINED: <что конкретно делаем, объём, откуда берём реализацию>` — передается в основной AI как детализация задачи.
- `STOP` — вывести пользователю причину и завершить без кодогенерации, обновив `.claude\rules\common-mistakes.md` причиной запрошенной пользователем.

## Слепой ревью

Вызвать Agent с model: "sonnet"
- [ ] **без описания задачи пользователя**
- [ ] передать:
- сгенерированный код
- `.claude/rules/red-lines.md`
- `.claude/rules/architecture.md`
- `.claude/rules/common-mistakes.md`
- `.claude/REFERENCE.md`
- Исключительные явные подтверждения пользователем (если они были запрошены Скептик-агентом)
- Намеренное явное указание пользователя для обхода red-lines в этом конкретном запросе.

Ожидаемый ответ агента:
- `OK`.
- `ASKED` — создание интерактивной формы по каждому вопросу в ходе критического обнаруженного нарушения.
