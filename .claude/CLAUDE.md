Проект: Скрипт развертывания скрытого прокси на удаленном сервере

---

## Модель общения

АИ и пользователь не имеют прямого доступа к локальным репозиториям друг друга.
Единственная точка синхронизации — удалённый репозиторий.

- Пользователь вносит правки локально → коммит → пуш в удалённый репо.
- АИ вытягивает удалённый репо → генерирует код в нужном формате → пользователь применяет локально.
- АИ **не может пушить**. Репо для АИ — только на чтение.

## Цикл работы

### Каждое сообщение:

1. **Синхронизация** — хук автоматически вытягивает удалённый репо (`REPO_SYNC` в system-reminder).
2. **Загрузка документации** — обязательно прочитать:
   - `.claude/rules/red-lines.md`
   - `.claude/rules/common-mistakes.md`
   - `.claude/REFERENCE.md` — **глобальный справочник проекта**: все компоненты, шаблоны конфигов, порты, переменные. Читать всегда.
   - `.claude/tasks/<N-активная-задача>.md` — **единственный task-файл для этого чата**

   > **REFERENCE.md vs ссылки в задаче:**
   > `REFERENCE.md` — знания об всём проекте (что есть, как устроено).
   > `## Ссылки` в task-файле — внешняя документация только для текущей задачи (официальные доки, репо компонента).
   > Если есть противоречие — спроси у пользователя приоритет ссылок.
3. **Скептик-Агент** — запустить до любой кодогенерации.
4. **Генерация кода** — только в границах активной задачи.
5. **Слепой ревью** — передать патч агенту-ревьюеру.
6. **Финал** — предоставить код пользователю (способ 1 или 2).

После этого: пользователь применяет локально → коммит → пуш → следующий цикл.

## Генерация кода

### Способ 1 — полный файл
Новый файл **или** более двух правок на файл:
- вывести файл целиком в блоке кода
- перед блоком кода — путь к файлу

### Способ 2 — замена фрагмента
До 5 строк, не более двух правок на файл:
- первый блок — старый код
- второй блок — новый код
- перед блоком — путь к файлу и номер строки

### Способ 3 — патч (строго по запросу)
Файл генерируется и отправляется через чат.
- Описание коммита: тэги [task-xxx] [bug-xxx] [ref-xxx]
- Себя указывать как контрибьютера.

## Скептик-Агент

Вызвать Agent с model: "haiku" **до любой кодогенерации**.
- [ ] **атакует задачу — не помогает**
- [ ] передать: задачу пользователя, `.claude/tasks/<N-активная-задача>.md`.claude/REFERENCE.md`, `.claude/rules/architecture.md`, код файлов которые будут затронуты

Атаки:
1. **Нужно ли это вообще?** — ищет готовое решение по документации приложений.
2. **Ты просишь то, что хочешь?** — запрашивает явную причину.
3. **Где граница?** — предлагает MVP ≤ 30 строк.

Всегда выводит интерактивную asked-форму и останавливает выполнение:
```
questions: [{"header":"","multiSelect":false,"options":[{"label":"","description":""}],"question":""}]
```

Ожидаемый ответ:
- `REFINED: <детализация>` — использовать вместо оригинала.
- `STOP` — завершить, обновить `.claude/rules/common-mistakes.md`.

## Слепой ревью

Вызвать Agent с model: "sonnet"
- [ ] **без описания задачи пользователя**
- [ ] передать: патч, `.claude/rules/red-lines.md`, `.claude/rules/architecture.md`, `.claude/rules/common-mistakes.md`, `.claude/REFERENCE.md`, `.claude/tasks/<N-активная-задача>.md`

Ответ: `OK` или `ASKED` с интерактивной формой.
