# [2] Проверка переменных

## Задача
Проверить корректность введённых пользователем переменных перед запуском установки.

## Цель
Предотвратить падение скрипта на середине установки из-за некорректных или незаполненных параметров.

## Обоснование
Мягкая валидация — использовать минимально рабочую конфигурацию и сообщить что будет/не будет установлено. Жёсткий выход — только для критичных ошибок, без которых сервис не запустится.

## Предусловия
- Блок переменных объявлен (задача 1)

## Результат
- Блок валидации добавлен в скрипт
- При запуске выводится сводка что будет установлено

## Связанные задачи
| Задача | Параметры |
|---|---|
| [1 — Переменные](1-variables.md) | источник всех проверяемых переменных |

## Детали

### Жёсткие проверки (exit 1)
| Условие | Сообщение |
|---|---|
| `INSTALL_TG=1` и `TG_API_ID` пусто | `[ERR] INSTALL_TG=1 но TG_API_ID не заполнен` |
| `INSTALL_TG=1` и `TG_API_HASH` пусто | `[ERR] INSTALL_TG=1 но TG_API_HASH не заполнен` |
| `NODE_DOMAIN` = `"example.com"` | `[ERR] NODE_DOMAIN не изменён (example.com)` |
| Secret в `TELEMT_CREDS` не соответствует 32 hex-символам | `[ERR] Невалидный telemt secret: <alias>` |

### Предупреждения (продолжить выполнение)
| Условие | Сообщение |
|---|---|
| Элемент `FP_CREDS` содержит `REPLACE_ME` | `[WARN] FP_CREDS содержит незаполненный пароль` |
| Элемент `TELEMT_CREDS` содержит `REPLACE_ME` | `[WARN] TELEMT_CREDS содержит незаполненный secret` |

### Итоговый вывод (после всех проверок)
Вывести сводку: что **будет** установлено и что **пропущено**.

```
Будет настроено:
  [x] Base system  (hostname, ufw, fail2ban)
  [x] telemt
  [x] Caddy
  [ ] .NET 10 backend        (INSTALL_DOTNET=0)
  [ ] Telegram Local Server  (INSTALL_TG=0)
```
